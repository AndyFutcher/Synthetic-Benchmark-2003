Firewalls & SynthMark XP.

 

See Also: Dedicated Servers, Back to Main.

 

Firewalls are becoming much more popular these days with security risks all round. People who have firewalls installed have to treat SynthMark XP just like a computer game, all internet applications use a unique port number to communicate with the servers on the internet. Firewalls generally block all the ports on your computer except for a few that you use.

 

The use of a firewall is especially important for SynthMark XP dedicated servers because you won’t be there while someone my try and illegally access your computer.

 

Firewalls are also especially needed for a corporate environment. If you need to use SynthMark XP across a office network you may have to speak to your system administrator and ask that the port number 27960 be released from the firewall protection list.

 

Port usage example:

Port Number

Usage

21

FTP protocol uses this, when downloading and uploading via a FTP server (File Transfer Protocol).

25

Email support for sending emails via your SMTP server.

110

Email support for receiving emails via your POP3 account server.

80

HTTP web page browsing with Internet Explorer or Netscape Navigator.

So likewise SynthMark XP needs it own port to be freed from your firewalls protection to access other computers across the network.

27960

This is the default port number is used for SynthMark XP and connecting via the NETMark system.

0-65536

You can specify any port number for SynthMark XP to use, including the ones above like 21 (FTP), but then you and your clients wont be able to access FTP servers while you are running SynthMark XP, and so forth.

 

The Windows XP firewall does not support port release customizations because it is quite a basic firewall (Designed for dummies). Even though it works well you cannot release the port number 27960, so we suggested the following…

 

·          You may either disable the firewall. (This is fine for Dial Up users).

·          Or use another firewall that does support port release, like McAfee personal firewall.

·          Use one of the other port numbers and sacrifice a connectivity system while using SynthMark XP.

 

Note: Do not use port 80 for your SynthMark XP server on the internet!
