The CPU is one of the more common benchmarking types, and indicates performance directly related to the performance of your computers main processor(s). This benchmark is spilt into two different benchmarking types, unique to SynthMark XP.

 

The Benchmarking Types:

 

·          The first benchmark is the Cache & FSB, this benchmark is effected more by the performance of your CPU’s cache memory and front side bus. It does this by issuing the CPU with redundant operations to allow the processor’s cache memory and branch predication to better influence the scores result.

 

·          The second benchmarking type is the RAM & Chipset benchmark. This benchmark has been optimized to use much bigger pieces of data than the cache memory can hold, forcing your CPU to use more of the RAM to hold vital information for the CPU. This speed is also affected by the chipsets performance to as the medium between the CPU and the RAM.

 

Understanding the CPU Benchmarks:

RAM stands for Random Access Memory and is much, much slower than the Cache memory of the CPU, which in modern times is all built-into the processor. RAM is upgradeable and it’s important to have ample the RAM otherwise your software programs have to start using the hard drive as temporary RAM, and hard drives are even slower than RAM.

 

The two benchmarks are totally different from each other but both play a huge role in common programs and are therefore very relative when it comes to everyday computing, and will most likely be the most common benchmark for online competitions.
